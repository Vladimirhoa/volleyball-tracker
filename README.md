# 🏐 Volleyball Tracker
Индивидуальный дневник волейбольных матчей. Приложение позволяет фиксировать результаты игр, вести статистику и хранить видеозаписи (загруженные напрямую или скачанные из VK) с автоматическим сжатием для экономии места на сервере.

---

## 🚀 Быстрый старт (Deployment)
Приложение полностью контейнеризировано с помощью Docker. Все необходимые зависимости (Python, PostgreSQL, Redis, FFmpeg) устанавливаются и настраиваются автоматически.

### 1. Системные требования
* **Docker** и **Docker Compose**

* **Git**

### 2. Клонирование репозитория
```bash
git clone https://github.com/Vladimirhoa/volleyball-tracker.git
cd volleyball-tracker
```

### 3. Настройка окружения
Создайте в корне проекта файл `.env` и укажите в нем свои параметры:
```bash
nano .env
```
Пример содержимого:
```
# Путь к директории для хранения видео на хост-машине
MEDIA_PATH=/mnt/storage/volleyball_media


# Параметры базы данных (будут созданы автоматически при первом запуске)
DB_NAME=volley_db
DB_USER=volley_admin
DB_PASSWORD=придумайте_сложный_пароль

# Доверенные адреса (ссылки на ваш сайт через запятую)
# Необходимо для работы кнопок «Поделиться» и защиты CSRF
CSRF_TRUSTED_ORIGINS=https://v-tracker.ru,https://www.v-tracker.ru,http://45.141.102.215:8080   
```
***Примечание: Переменные `DB_NAME`, `DB_USER` и `DB_PASSWORD` используются как для инициализации контейнера PostgreSQL, так и для подключения Django к базе.***

### 4. Запуск приложения
Соберите образы и запустите контейнеры в фоновом режиме:
```bash
sudo docker-compose up -d --build
```
После запуска сайт будет доступен по адресу вашего сервера на порту 8000.

### 5. Создание учетной записи администратора
Для доступа в панель управления и админку Django создайте суперпользователя:
```bash
sudo docker exec -it volleyball_tracker python manage.py createsuperuser
```

## 🛠 Технологический стек
* **Django 6.0**: Основной веб-фреймворк.

* **PostgreSQL 15**: Надежное хранилище данных.

* **Celery + Redis**: Очередь задач для фоновой обработки и планировщик задач.

* **FFmpeg**: Инструмент для автоматического сжатия видео в формат 720p (libx264, CRF 28).

* **yt-dlp**: Библиотека для скачивания видео из VK.

* **Nginx**: Веб-сервер и Reverse Proxy для отдачи статики и медиа-файлов.

---

## 📦 Особенности реализации
* **Chunked Upload**: Загрузка больших видеофайлов реализована через последовательную отправку чанков по 10 МБ, что обеспечивает стабильность при нестабильном соединении.

* **Database Healthcheck**: Использование механизма проверок состояния (healthcheck) гарантирует, что приложение начнет работу только после полной готовности базы данных PostgreSQL.

* **Background Processing**: Все тяжелые операции (сжатие видео, скачивание из внешних источников) вынесены в фоновые задачи Celery, чтобы не блокировать работу пользователя с интерфейсом.

* **Auto-cleanup**: Система автоматически очищает временные файлы и незавершенные загрузки каждую ночь.
